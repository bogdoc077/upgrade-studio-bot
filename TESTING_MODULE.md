# Модуль тестування автоматичних сценаріїв

## Опис

Новий розділ "Тестування" в адмін-панелі дозволяє запускати автоматичні сценарії вручну для перевірки їх роботи на конкретних користувачах.

## Можливості

### Доступні тести:

1. **Попередження про списання (7 днів)**
   - Надсилає нагадування що через 7 днів відбудеться автоматичне списання
   - Вимагає активної підписки
   - Імітує автоматичний сценарій `check_upcoming_payments`

2. **Закінчення підписки**
   - Імітує закінчення підписки користувача
   - Тимчасово скидає доступи (subscription_active, joined_channel, joined_chat)
   - Надсилає пропозицію оформити підписку знову
   - Через 5 секунд відновлює оригінальні значення
   - Імітує сценарій `check_expired_subscriptions`

3. **Нагадування про призупинену підписку**
   - Надсилає нагадування користувачу з призупиненою підпискою
   - Показує скільки днів залишилось до закінчення доступу
   - Вимагає призупиненої підписки (subscription_paused = true)

4. **Нагадування про приєднання**
   - Надсилає нагадування про необхідність приєднатися до каналів
   - Підходить для користувачів які мають підписку, але не приєдналися

## Як користуватися

### 1. Вибір користувача
- Введіть ім'я, username або Telegram ID в поле пошуку
- З випадаючого списку оберіть потрібного користувача
- Система покаже підтвердження вибору з Telegram ID

### 2. Запуск тесту
- Натисніть кнопку "Запустити тест" біля потрібного сценарію
- Користувач отримає реальне повідомлення в Telegram
- Результат виконання відобразиться під описом тесту

### 3. Результати
- ✅ Зелений блок = тест успішно виконано
- ❌ Червоний блок = виникла помилка
- Детальний опис результату у повідомленні

## Технічна реалізація

### Backend (API)

**Файл:** `api/server.py`

Додано 4 нові ендпоінти:

```python
POST /api/testing/check-upcoming-payment
POST /api/testing/expired-subscription
POST /api/testing/paused-subscription-reminder
POST /api/testing/join-reminder
```

Всі ендпоінти:
- Захищені токеном (`dependencies=[Depends(verify_token)]`)
- Приймають `telegram_id` у body запиту
- Перевіряють існування користувача
- Виконують відповідні дії
- Повертають результат у форматі `{success: bool, message: string}`

### Frontend (Admin Panel)

**Файл:** `admin-panel/src/app/testing/page.tsx`

Компонент включає:
- Пошук користувачів в реальному часі
- Autocomplete dropdown з результатами
- Картки для кожного тестового сценарію
- Відображення результатів тестів
- Обмеження на запуск тестів (потребує вибору користувача)

**Файл:** `admin-panel/src/components/Sidebar.tsx`

Додано:
- Іконку `BeakerIcon` для Testing
- Новий пункт меню "Тестування" (`/testing`)

## Безпека

- Всі API запити захищені JWT токеном
- Тести надсилають справжні повідомлення - використовувати обережно
- Деякі тести тимчасово змінюють стан користувача (відновлюється автоматично)
- Рекомендується тестувати на власному акаунті або тестових користувачах

## Приклади використання

### Тестування нагадування про оплату
1. Оберіть користувача з активною підпискою
2. Запустіть тест "Попередження про списання"
3. Користувач отримає повідомлення в боті
4. Перевірте що кнопка "В головне меню" працює

### Тестування закінчення підписки
1. Оберіть користувача (будь-який стан підписки)
2. Запустіть тест "Закінчення підписки"
3. Користувач втратить доступи на 5 секунд
4. Отримає пропозицію оформити підписку
5. Через 5 секунд доступи відновляться автоматично

## Розширення функціоналу

Для додавання нового тесту:

1. Додайте API ендпоінт у `api/server.py`:
```python
@app.post("/api/testing/your-new-test", dependencies=[Depends(verify_token)])
async def test_your_scenario(data: dict):
    telegram_id = data.get('telegram_id')
    # Ваша логіка тесту
    return {"success": True, "message": "Test completed"}
```

2. Додайте сценарій у `TEST_SCENARIOS` в `testing/page.tsx`:
```typescript
{
  id: 'your_test',
  name: 'Назва тесту',
  description: 'Опис що робить цей тест',
  endpoint: '/api/testing/your-new-test',
  requiresActiveSubscription: false // опціонально
}
```

3. Тест автоматично з'явиться в UI з усім функціоналом

## Логування

Всі тести логуються в Python logger:
- Успішні запуски: `logger.info()`
- Помилки: `logger.error()`
- Можна переглянути в розділі "Логи системи"

## Обмеження

- Тести надсилають реальні повідомлення користувачам
- Деякі тести вимагають певного стану підписки
- Затримки між повідомленнями згідно з Telegram API обмеженнями
- "Закінчення підписки" відновлює стан через 5 секунд автоматично
